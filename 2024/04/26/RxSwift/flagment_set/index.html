weak 与 unowned的异同

weak 和 unowned 都是 Swift 中用来避免循环引用的关键字，尤其在闭包中引用类的实例时。它们都能用来定义引用而不会增加引用计数，因此帮助打破循环引用。但它们之间有一些重要的区别：

  • weak:
    • weak 引用必须声明为可变 (var)，因为它的引用可以在运行时改变（被设置为 nil）。
    • weak 引用永远都是可选类型，这意味着您必须在使用时解包（可通过可选链或者强制/条件解包）。
    • 当引用的对象被销毁时，weak 引用会自动变为 nil。
    • 使用 weak 的场合是：您预计引用的对象在引用的生命周期内可能会被释放，此时使用 weak 更加安全，因为引用会变成 nil，可以防止野指针错误。
  • unowned:
    • unowned 引用可以是常量（let）或变量（var），因为它不会在所引用的对象被销毁后变成 nil。
    • unowned 引用不是可选类型，因此使用时不需要解包。
    • 当引用的对象被销毁后，尝试访问 unowned 引用将触发运行时错误（崩溃）。
    • 使用 unowned 的场合是：您确信引用的对象在引用生命周期内都不会被释放，因此在使用引用时不需要考虑它成为 nil 的情况。发生在两个生命周期相互关联时，比如父子关系的对象，其中父对象拥有子对象，而子对象只是无主（unowned）引用父对象。

在选择使用 weak 还是 unowned 时，应该考虑引用的对象是否可能在闭包的生命周期内变为 nil。如果可能的话，使用 weak 并且在闭包内安全地处理可能的 nil 值。如果不可能的话，或者如果 nil 出现时程序逻辑上有缺陷（也就是说，如果引用对象不存在程序无法正常工作），那么选用 unowned 是合适的

这里使用 unowned 因为 asHTML 闭包是 HTMLElement 类的一部分，只有当 HTMLElement 实例存在时，这个闭包才有意义。如果 HTMLElement 实例被释放了，闭包就没有存在的意义了，因此不需要考虑闭包持有的 unowned self 会变成 nil 的情况。当然，如果实例销毁后还试图使用这个闭包，就会运行时错误。
